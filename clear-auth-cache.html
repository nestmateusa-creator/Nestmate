This tool will help you completely clear all authentication data and resolve the auto-redirection issue.

Step 1: Sign Out from Both Firebase Projects

This will clear the authentication state from both old and new Firebase projects.

Step 2: Clear Browser Storage

This will clear localStorage, sessionStorage, and other cached data.

Step 3: Clear IndexedDB

This will clear any IndexedDB data that might be storing authentication tokens.

Step 4: Clear All Cache (Nuclear Option)

This will attempt to clear everything. Use this if the above steps don't work.

Step 5: Manual Browser Cache Clear

You should also manually clear your browser cache:

  • Chrome/Edge: Press Ctrl+Shift+Delete, select "All time", check all boxes, click "Clear data"
  • Firefox: Press Ctrl+Shift+Delete, select "Everything", click "Clear Now"
  • Safari: Press Cmd+Option+E to clear cache

Next Steps After Clearing Cache:

  1. Close all browser tabs with your application
  2. Open a new incognito/private window
  3. Go to signin.html
  4. Try creating a new account
  5. If you still get redirected to the old account, try a different browser

Firebase Security Rules

Don't forget to set up proper Firestore security rules in your Firebase Console to fix the "Permission denied" errors. See the firebase-security-rules.md file for instructions.
